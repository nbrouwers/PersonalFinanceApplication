## ADDED Requirements

### Requirement: Ubiquitous language is reviewed during proposal
The development workflow SHALL require checking and updating the ubiquitous language document as part of every OpenSpec proposal.

#### Scenario: New domain terms are captured before implementation
- **WHEN** a proposal introduces new domain concepts or renames existing ones
- **THEN** those terms SHALL be added to the ubiquitous language document before the apply phase begins

#### Scenario: User can review UL changes alongside proposal
- **WHEN** the proposal, design, specs, tasks, and UL updates are all created
- **THEN** execution SHALL pause for user review before the apply phase
